# 数据分析 Agent 系统功能手册

> 同步日期：2026-04-02（已按当前代码实现更新）

## 一、启动与入口

```bash
langgraph dev
```

- Studio：`http://localhost:2024`
- 图入口：`graph.py:data_agent`
- FastAPI（可选）：`python scripts/run_api.py`（默认 `:8000`）

## 二、运行时架构（当前实现）

```text
用户请求
  -> SupervisorAgent(create_react_agent)
    -> Middleware 组合层
      - FilesystemMiddleware
      - TodoListMiddleware
      - SubAgentMiddleware
      - ContextEditing / ToolAuth / ToolCache / ToolRuntime / Summarization
    -> 子 Agent 派发
      - sql_specialist
      - data_analyst
      - visualization_specialist
```

补充：
- SQL 生成与执行已解耦：`sql_generator_agent` 负责生成，`sql_specialist` 负责校验/执行/重试。
- 上下文治理已落地：`context_guardian_agent` + `context_compact_gate`（LLM 主路径，规则兜底）。
- SQL 执行前 HITL 已落地：支持 `approve/edit/reject`。

## 四、SQL 专家执行链（已实现）

```text
context_compact_gate()
  -> _collect_table_context()  # 语义层检索
  -> generate_sql_via_agent()  # llm_skill 严格 JSON 契约
  -> validate_sql_for_executor()
  -> [可选] _hitl_interrupt()  # enable_hitl=true
  -> sql_inter 执行
  -> 失败时 repair_sql_on_error() 并最多重试一次
```

### 4.1 SQL 契约

生成目标：

```json
{"sql_query": "SELECT ..."}
```

执行前校验：
- 仅允许 `SELECT` 或 `WITH` 开头
- 禁止 `INSERT/UPDATE/DELETE/DROP/CREATE/ALTER/TRUNCATE`

### 4.2 HITL（SQL 执行前人工确认）

中断载荷：

```json
{
  "stage": "pre_execution",
  "sql_query": "SELECT ...",
  "schema_digest": "...",
  "task_goal": "..."
}
```

恢复决策：
- `approve`：按原 SQL 执行
- `edit`：使用 `edited_sql`，并重新走 SQL 校验
- `reject`：终止执行并返回拒绝原因

## 五、上下文压缩与槽位注入

`SubAgentMiddleware` 内部维护四类槽位：
- `decision_summary`
- `schema_summary`
- `sql_summary`
- `error_summary`

机制：
- 优先 `_llm_summarize_context()` 产出结构化摘要
- 失败时 `_rule_summarize_fallback()`
- `_route_summary_to_slots()` 按槽位写入，单槽位保留最近 3 条
- `_compose_runtime_prompt_from_slots()` 注入 SQL 生成 prompt

快照文件：
- `/files/context_snapshot_{thread_id}.md`

## 六、语义层能力

- 首次 SQL 任务时自动构建语义层缓存（表名、字段、样例、metadata）
- 向量模型：`BAAI/bge-m3`
- 检索顺序：向量检索 -> token 匹配 -> 前 top-k 兜底
- 可手工刷新：`rebuild_data_semantic_layer()`

## 八、配置项（新增重点）

`config/settings.py` 关键字段：
- `enable_hitl: bool = False`
- `hitl_timeout_seconds: int = 300`
- `checkpointer_backend: str = "memory"`（`memory/sqlite`）
- `checkpointer_path: Optional[str]`

说明：`graph.py` 中 `get_checkpointer()` 支持 sqlite，不可用时自动回退 `MemorySaver`。
