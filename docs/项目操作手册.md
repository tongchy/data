## 环境准备

### 系统要求

- **操作系统**: Linux / macOS / Windows
- **Python 版本**: 3.9+ （推荐 3.11）
- **包管理**: pip 或 conda
- **数据库**: MySQL 8.0+
- **互联网**: 需要访问 SiliconFlow API 分析

### 依赖检查

运行以下命令检查环境是否就绪：

```bash
python check_env.py
```

输出应显示所有依赖都已安装。

## 项目安装

### 方式一：标准安装（推荐）

```bash
# 1. 克隆或进入项目目录
cd /workspaces/2026_03_30

# 2. 创建虚拟环境（可选，推荐）
```


## 配置指南

### 环境变量配置

#### 1. 复制配置文件

```bash
cp .env.example .env
```

#### 2. 编辑 `.env` 文件

```bash
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=alarm

# 模型配置（SiliconFlow DeepSeek）
MODEL_API_KEY=your_siliconflow_api_key
MODEL_BASE_URL=https://api.siliconflow.cn/v1
MODEL_MODEL_NAME=deepseek-ai/DeepSeek-V3
MODEL_TEMPERATURE=0.1
MODEL_MAX_TOKENS=4096

# LangSmith 配置（可选，用于监控）
LANGSMITH_API_KEY=your_langsmith_api_key
LANGSMITH_PROJECT=your_project_name

# 应用配置
DEBUG=false
LOG_LEVEL=INFO
```

#### 3. 配置说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DB_HOST` | localhost | MySQL 服务器地址 |
| `DB_PORT` | 3306 | MySQL 端口 |
| `DB_USER` | root | 数据库用户名 |
| `DB_PASSWORD` | - | 数据库密码（必须设置） |
| `DB_NAME` | alarm | 默认数据库名称 |
| `MODEL_API_KEY` | - | SiliconFlow API 密钥（必须设置） |
| `MODEL_TEMPERATURE` | 0.1 | 模型创意度（0.0-2.0） |
| `MODEL_MAX_TOKENS` | 4096 | 最大输出 token 数 |
| `DEBUG` | false | 调试模式（用于开发） |
| `LOG_LEVEL` | INFO | 日志级别（DEBUG/INFO/WARNING/ERROR） |

### 数据库初始化

```bash
# 方式一：使用脚本初始化
python scripts/setup_db.py

# 方式二：手动连接 MySQL，执行以下 SQL
mysql -u root -p alarm < scripts/init_db.sql
```


```bash

result = asyncio.run(agent.invoke("分析设备告警趋势", thread_id="test_1"))
print(result)
```

## API 使用

### API 服务基本信息

- **基础 URL**: `http://localhost:8000`
- **API 文档**: `http://localhost:8000/docs` (Swagger UI)
- **文档替代**: `http://localhost:8000/redoc` (ReDoc)

### 核心 API 端点

#### 1. 发送聊天消息

**端点**: `POST /api/chat`

**请求体**:
```json
{
  "message": "分析最近7天的告警趋势",
  "thread_id": "session_001",
  "context": {
    "user_id": "user_123",
    "role": "analyst"
  }
}
```

**响应**:
```json
{
  "messages": [
    {
      "role": "user",
      "content": "分析最近7天的告警趋势"
    },
    {
      "role": "assistant",
      "content": "我来为您分析最近7天的告警数据...",
      "tool_calls": [
        {
          "name": "query_tool",
          "args": {"sql": "SELECT COUNT(*) FROM alarms WHERE ..."}
        }
      ]
    }
  ],
  "status": "completed",
  "thread_id": "session_001"
}
```

#### 2. 流式聊天响应

**端点**: `POST /api/chat_stream`

返回 Server-Sent Events (SSE) 流，适合前端实时显示。

```bash
curl -X POST http://localhost:8000/api/chat_stream \
  -H "Content-Type: application/json" \
  -d '{
    "message": "分析告警数据",
    "thread_id": "session_001"
  }' \
  -N  # 不缓冲输出
```

#### 3. 获取工具列表

**端点**: `GET /api/tools`

**响应**:
```json
{
  "tools": [
    {
      "name": "query_tool",
      "description": "执行 SQL 数据查询",
      "schema": {...}
    },
    {
      "name": "extract_tool",
      "description": "数据提取和转换",
      "schema": {...}
    }
  ]
}
```

#### 4. 获取健康检查

**端点**: `GET /api/health`

**响应**:
```json
{
  "status": "healthy",
  "version": "2.0.0",
  "database": "connected",
  "model": "ready"
}
```

## 常见问题

### Q1: 如何修改数据库连接？

**A**: 编辑 `.env` 文件中的数据库变量：

```bash
DB_HOST=your_host
DB_PORT=3306
DB_USER=your_user
DB_PASSWORD=your_password
DB_NAME=your_database
```

### Q2: 如何切换 LLM 模型？

**A**: 修改 `.env` 中的模型配置：

```bash
MODEL_API_KEY=your_new_api_key
MODEL_BASE_URL=http://your_model_server
MODEL_MODEL_NAME=your_model_name
```

或在代码中：

```python
from config.settings import Settings
from models.llm import create_llm

settings = Settings()
settings.model.model_name = "new_model"
llm = create_llm(settings)
```

### Q3: 如何添加自定义工具？

**A**: 参考 [代码文件手册 - tools 模块](#)。基本步骤：

```python
# 1. 创建工具类
from tools.base import BaseCustomTool, ToolResult

class MyTool(BaseCustomTool):
    name = "my_tool"
    description = "My custom tool"
    
    def _execute(self, **kwargs) -> ToolResult:
        return ToolResult(success=True, data=result)

# 2. 注册工具
from tools.registry import registry
registry.register(MyTool())

# 3. 在 supervisor 中使用
```

### Q4: 如何调试 Agent 执行流程？

**A**: 使用 LangSmith 进行追踪（需要配置 API Key）：

```bash
# 1. 设置 LangSmith 环境变量
export LANGSMITH_API_KEY=your_key
export LANGSMITH_PROJECT=your_project

# 2. 启动服务
python scripts/run_api.py

# 3. 访问 https://smith.langchain.com 查看执行流
```

或使用调试模式运行：

```python
import logging
logging.basicConfig(level=logging.DEBUG)

# 然后运行 Agent
agent.invoke(message)
```

### Q5: API 返回 401 错误？

**A**: 检查以下几点：

1. 验证 MODEL_API_KEY 是否正确设置
2. 检查 API Key 是否过期
3. 查看 SiliconFlow API 额度是否充足
4. 查看日志文件了解详细错误信息

```bash
tail -f logs/app.log
```

### Q6: 如何访问对话历史？

**A**: 使用 `thread_id` 恢复对话：

```python
# 使用相同的 thread_id 继续对话
result = await agent.invoke(
    "继续之前的分析",
    thread_id="session_001"
)

# Agent 会自动读取历史消息
```
