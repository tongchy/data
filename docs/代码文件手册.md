## 2. 入口与编排

### 2.1 graph.py

职责：
- 提供 LangGraph 入口 `data_agent`
- 构建 `store` 与 `checkpointer`
- 暴露 `get_supervisor_agent/get_tools/get_agent_status`

关键点：
- `get_checkpointer()` 支持 `memory/sqlite`
- sqlite 失败自动回退 `MemorySaver`

### 2.2 agents/supervisor.py

职责：
- 统一组装中间件、工具与子 Agent
- 提供 `invoke()` 与 `resume()`
- 维护 `thread_states`、`runtime_graphs`

新增能力：
- `resume(thread_id, payload)` 支持 HITL 恢复
- 提示词包含“纯文本任务优先 llm_skill”

## 3. 子 Agent 与中间件

### 3.1 agents/subagents/

- `sql_agent.py`：SQL 专家定义
- `sql_generator_agent.py`：SQL 生成子 Agent（生成职责）
- `context_guardian_agent.py`：上下文治理子 Agent
- `data_analysis_agent.py`：数据分析子 Agent
- `visualization_agent.py`：可视化子 Agent

### 3.2 middleware/subagent.py

核心方法：
- `generate_sql_via_agent`
- `validate_sql_for_executor`
- `repair_sql_on_error`
- `context_compact_gate`
- `_llm_summarize_context`
- `_route_summary_to_slots`
- `_compose_runtime_prompt_from_slots`
- `_hitl_interrupt`
- `_hitl_resume_handler`
- `_run_sql_specialist`

职责拆分：
- SQL 生成：`sql_generator_agent + llm_skill`
- SQL 执行：`sql_inter`
- 失败修复：`repair_sql_on_error`（一次重试）
- 上下文治理：LLM 摘要主路径 + 规则兜底

## 4. API 文件

### 4.1 api/main.py

- 注册路由：`health/chat/resume/tools`
